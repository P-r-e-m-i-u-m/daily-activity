# Incident Report: INC-0017
## SEV-2: Rate Limiter Degradation — 38 minutes Outage

**Date:** 2026-04-26

## Executive Summary

Rate limiter TTL misconfiguration caused 890 legitimate requests to be incorrectly blocked with 429 errors. 340 unique IPs were affected.

## Timeline (All times UTC)

| Time | Event |
|-|-|
| 11:00 | Rate limiter TTL misconfiguration deployed |
| 11:08 | Alert fires: 429 error rate above 2 percent |
| 11:12 | Engineer identifies TTL bug in sliding window |
| 11:22 | Hotfix deployed — incorrect keys flushed |
| 11:38 | Error rate returns to baseline — incident resolved |

## Root Cause Analysis

A bug in the sliding window implementation set the Redis key TTL in seconds instead of milliseconds, causing keys to expire 1000x faster than intended.

## Resolution

Identified and flushed incorrectly-configured Redis keys. Deployed hotfix correcting TTL unit. Added integration test verifying actual Redis TTL behavior.

## Lessons Learned

1. Load testing thresholds must reflect production concurrency levels
2. All infrastructure maintenance must follow the change management runbook
3. Integration tests must use real dependencies not just mocks
4. Security hardening changes must be reflected in staging environment
